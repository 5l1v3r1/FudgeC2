Welcome to FudgeC2

Start by creating a new campaign, and allocating access to any other team members.

To generate an implant select the desired campaign and select 'New Implant', configuring the required fields.

Further info

Campaigns are broken down into 3 functional areas, implants templates, and stagers, and active listeners. An implant template is responsible for creating the base information such as:

  • Common name
  • Callback URL
  • Communication channels
  • Beacon and initial communication delays

After an implant template has been created FudgeC2 will automatically generate stagers which can be used to gain initial access.

  • A stager can be used numerous times, once a stager communicates with the C2 server a the full implant will be delivered.
  • Each implant that is delivered will generate it's own unique implant reference.

Once an implant call been deployed it will be given a unique common name based on the implant templates common name. It can now be interacted with as the full implant using the campaign main page. Once a common has been registered with FudgeC2 it will await pick up from the implant the next time it beacons back. Each new common will add to the queue.

When a common returns data to the C2 it will be timestamped based on the time C2 server receives the information.

For full documentation see the README file, or visit the GitHub page: https://github.com/Ziconius/FudgeC2
